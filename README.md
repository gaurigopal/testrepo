1. git clone <repository-url>
   - Copies a remote repository from GitHub to your local machine.
   - Used only once when you first download the repository.

2. git branch
   - Displays the available branches.
   - Shows the current active branch with an asterisk (*).

3. git add .
   - Stages all new and modified files.
   - Prepares changes for committing.

4. git commit -m "message"
   - Saves the staged changes to the local Git repository.
   - The message describes what changes were made.

5. git push
   - Uploads committed changes from the local repository to the remote repository (GitHub).
     
6. git pull

Typical Workflow:
git clone → modify files → git add . → git commit -m "message" → git push
